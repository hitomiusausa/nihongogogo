# 改修記録

## 2026-07-17 大規模改修（クレーム対応＋機能改善）

公開ページ: https://hitomiusausa.github.io/nihongogogo/
本番実行: GitHub Actions `daily-update.yml`（毎日23:00 UTC＝朝8時JST）→ GitHub Pages（`docs/`）

### きっかけ（利用メンバーからのクレーム）

1. 「同じニュースが複数出ている」— 同一プレスリリースが最大4カード並んでいた
2. 「リンクを辿っても情報がない」— 愛知県の補助金ページが404のまま毎日掲載され続けていた

### 原因と対策

**重複表示**
- 原因: タイトル指紋が記号の揺れ（～/〜・［］/【】・全角数字・切り詰め「..」・媒体名付記）で割れ、完全一致でしか束ねていなかった
- 対策: 指紋をNFKC正規化＋記号全除去に強化（スキーマv2でtitle_key全行再計算）。表示時は包含・最長共通部分文字列8割の近似判定で束ねる。年度違いの定例案件（令和7/8年度）と定型句を共有するだけの別告示は束ねない保護付き
- 束ねた他媒体の報道は捨てずにカード内「関連報道」リンクとして残す

**リンク切れ**
- 原因: リンク生死検査がどこにも存在せず、収集も詳細ページ取得失敗を素通ししていた
- 対策: `check-links`コマンド新設（日次実行に組込み）。404/410は**「リンク切れ（YYYY-MM-DD確認）」赤バッジ＋グレーアウトで一覧に残す**（隠さない・記録として保持）。ページ復活時は自動でバッジが外れる。収集時も404詳細ページは取り込まない
- 巡回先が消滅した「岐阜県 多文化共生補助金」ソースは削除（移転先HTMLなし・交付要綱PDFのみ現存）

### 機能改善（4案）

- **A カード情報の質**: Google Newsカードの概要（タイトルの焼き直しだった）を「掲載元: 媒体名」に置換。関連報道の束ね表示
- **B 収集ヘルス**: ソース別取得数・エラー・取得0件の巡回ページを`data/health.json`に保存し、サイトフッターと日次レポート末尾に表示（ソースの沈黙死に気づける）
- **C 締切UX**: 公募セクションを「締切近い順→締切なし→終了」で表示。`first_seen_at`列を追加（スキーマv3）し、今日初出のニュースに**「新鮮ニュース」バッジ＋フィルタ**
- **D 金額抽出**: 詳細ページから補助上限額・補助率を抽出し緑バッジ表示（「5,000千円」等は丸めなしで「500万円」に正規化）

### 開発メモ

- テスト: `python3 -m unittest discover -s tests`（61本）
- スキーマ移行は`PRAGMA user_version`ゲート（現行v3）。`title_key`は常に`title_fingerprint(title)`と一致させる
- `fetched_at`は毎日の再取得で更新される。「初出日」は`first_seen_at`
- 重複判定の閾値は「束ねてはいけない実例」（KP2MI別告示・年度違い）をテストに固定して裁定している。閾値を触るときは`tests/test_fetchers.py`のDuplicateTitleKeyTestを先に読むこと
